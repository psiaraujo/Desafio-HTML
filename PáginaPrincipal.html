Como nasceu o HTML?

(Voltar)

O HTML, ou HyperText Markup Language (Linguagem de Marcação de Hipertexto, em português), foi desenvolvido por Tim Berners-Lee, um cientista da computação britânico, enquanto trabalhava no CERN (Organização Europeia para a Pesquisa Nuclear) no início da década de 1990. A ideia era criar um sistema de documentação que permitisse a interconexão de documentos científicos.

Em 1991, Berners-Lee propôs o conceito de um "espaço de informação" global baseado em hipertexto, que se tornou a base da World Wide Web. No mesmo ano, ele criou o primeiro navegador web chamado "WorldWideWeb" (também conhecido como Nexus), que tinha a capacidade de visualizar páginas da web que eram formatadas usando a linguagem de marcação inicial que mais tarde se tornaria o HTML.

O HTML evoluiu ao longo do tempo com o envolvimento de várias pessoas e organizações, tornando-se uma parte fundamental da construção de páginas da web e da estrutura da World Wide Web. Desde então, o HTML passou por várias versões, cada uma introduzindo novos recursos e melhorias na linguagem. O desenvolvimento contínuo do HTML é supervisionado pelo World Wide Web Consortium (W3C) e pela Web Hypertext Application Technology Working Group (WHATWG).

Qual a relevância do HTML?

(Voltar)

O HTML (HyperText Markup Language) é fundamental para a criação e formatação de páginas web. Sua relevância é notável por várias razões:

Estruturação de Conteúdo:
O HTML fornece uma estrutura básica para o conteúdo de uma página web, definindo elementos como títulos, parágrafos, listas, links, imagens e outros componentes.

Interconexão de Documentos:
Através de links, o HTML permite a interconexão de documentos, possibilitando a navegação entre diferentes páginas e recursos na web.

Compatibilidade com Navegadores:
É uma linguagem padrão suportada por todos os principais navegadores, garantindo a consistência na exibição de conteúdo na web.

Base para Desenvolvimento Web:
O HTML é a base para o desenvolvimento web, muitas vezes combinado com CSS (Cascading Style Sheets) para estilos e JavaScript para interatividade.

Acessibilidade: A marcação semântica no HTML ajuda na criação de conteúdo acessível, o que é crucial para garantir que pessoas com diferentes habilidades possam interagir com as páginas web.

SEO (Search Engine Optimization):
Motores de busca utilizam a estrutura HTML para entender e indexar o conteúdo das páginas web, influenciando diretamente nas classificações de busca.

Desenvolvimento Responsivo:
O HTML5 introduziu recursos que facilitam a criação de páginas web responsivas, capazes de se adaptar a diferentes dispositivos e tamanhos de tela.

Padrões da Web:
O HTML é desenvolvido e mantido por organizações como o World Wide Web Consortium (W3C) e a Web Hypertext Application Technology Working Group (WHATWG), o que contribui para a padronização e interoperabilidade na web.

Em resumo, o HTML desempenha um papel central na construção da World Wide Web, proporcionando uma estrutura para a criação de documentos interativos e acessíveis. Sua importância é evidente na base de todos os desenvolvimentos web modernos.

Qual a estrutura básica do HTML?

(Voltar)

O documento HTML sempre inicia com o que chamamos de estrutura básica. Esta estrutura é quase que imutável. Sempre será dessa forma e você sempre, sempre começará seu HTML começando por esse código. Geralmente os editores como o Sublime Text já tem atalhos para iniciar os documentos HTMLs com essa estrutura, logo, você não precisa se preocupar em decorá-la, mas é bom que faça.

É possível compreender o documento em HTML de uma maneira muito simples, através de uma divisão de blocos das tags essenciais, conforme a a seguinte estrutura: Definição do documento (doctype) Cabeça (head) Corpo (body)

Doctype - Definindo o documento Uma coisa importante: SEMPRE deve existir o doctype, que é este código . O doctype não é uma tag HTML, mas uma instrução para o navegador e outros programas que podem ler seu site, que o código encontrado ali é um código HTML. Assim eles sabem o que fazer para mostrar seu site da melhor forma possível. Lembre-se: o doctype é OBRIGATÓRIO e deve ser sempre a PRIMEIRA LINHA do seu documento.

HEAD Contém informações que não são transpostas visivelmente para o usuário/leitor do documento. São dados implícitos, de uso e controle do documento: vinculação com outros arquivos, aplicação de lógica de programação de scripts e metadados. Na prática, todo o conteúdo do cabeçalho fica delimitado entre a abertura e fechamento tag head.

BODY Trata-se do documento em si, ou seja, a informação legível para o usuário/leitor do documento. É todo e qualquer texto que se deseja apresentar, assim como toda e qualquer forma de mídia de saída (imagens, sons, miniaplicativos embutidos, conteúdo multimídia, etc). Além disso, toda a apresentação de entrada de dados (formulários) também se aplica neste seção do documento. Na prática, o corpo do documento é delimitado pelo par de tags e . Este é o preceito básico que deve estar muito bem claro para você: onde as marcações se aplicam, e quais são os resultados deste modelo. Por exemplo: se vocês deseja informar conteúdo textual para saída legível ao usuário do seu sistema web, esta marcação deverá obrigatoriamente estar no bloco do corpo da página. Ainda: para definir qual o tipo de codificação da página (uma meta informação do documento), esta deve obrigatoriamente estar marcada no cabeçalho do mesmo documento. Dentro do elemento BODY sua estrutura de página terá os elementos semânticos da construção da sua página, onde serão declarados e identificados cabeçalhos, rodapé, conteúdo principal, etc.
